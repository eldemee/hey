# Revenue Prediction in Tabular Data: A Machine Learning Approach
This project is an assignment for my Advanced Analytics in Business course at KU Leuven, focused on predicting future consumer expending.

## 💻 Implementation and Purpose
Preprocessing, featurization, and data cleaning. This was the most time-consuming stage, with improvements achieved through empirical testing (see notebooks/01_preprocessing.ipynb). Initially, I analyzed each available variable and evaluated its usefulness based on the information it provides to the model. After selecting the most relevant variables, I cleaned the data and engineered new informative features.

The variables that best explain the variance in the regression model are:

prob_return: the predicted probability that a customer generates revenue
customer_revenue: the total spending of a customer
n_products: the number of products purchased
- Modeling. The project consists of two models (see notebooks/02_modeling.ipynb). The first is a classification model that computes prob_return, i.e., the predicted probability that a customer generates revenue. The second is a regression model, whose objective is to minimize the Mean Absolute Error (MAE). Both models are implemented using CatBoost.
Additionally, the regression model incorporates sample weighting based on the predicted probability of return. Observations with higher prob_return are assigned greater weight during training (𝑤=1+3*prob_return), which improves model performance. This approach prioritizes customers who are more likely to generate revenue, allowing the model to focus on the most economically relevant observations.
CatBoost is a gradient boosting algorithm based on decision trees, developed by Yandex. It is particularly well-suited for tabular data and has several advantages. First, it handles categorical variables natively through ordered target encoding, which reduces the need for extensive preprocessing and helps prevent target leakage. Second, it uses symmetric (oblivious) trees, which improve computational efficiency and reduce overfitting. Third, it incorporates ordered boosting, a technique designed to mitigate prediction shift and improve generalization performance.
- Tuning (see notebooks/03_tuning.ipynb). Hyperparameters for both models were optimized using grid search with cross-validation. For the classification model, the selection criterion was ROC-AUC, while for the regression model it was Mean Absolute Error (MAE). The tuning process focused on key CatBoost hyperparameters such as tree depth, learning rate, and number of iterations, aiming to balance predictive performance and overfitting.
- Validation (see notebooks/04_validation.ipynb). Model performance was evaluated using multiple metrics to assess both classification and regression tasks. For the classification model, ROC-AUC and Average Precision (PR-AUC) were used to measure discriminatory power, especially under class imbalance. For the regression model, performance was evaluated using Mean Absolute Error (MAE) and Root Mean Squared Error (RMSE) over the full dataset. Additionally, improvements were quantified relative to a baseline model in terms of MAE reduction.
To better understand feature contributions, feature importance analysis was conducted, allowing us to assess the explanatory power of each variable in the model.
## 📚 What I Learned
This project provided practical experience in applying machine learning techniques to tabular data, highlighting the importance of data preprocessing and feature engineering as key drivers of model performance. I learned how to design a two-stage modeling approach, combining classification and regression to better capture the underlying structure of the problem.
Additionally, I gained experience in hyperparameter tuning using cross-validation and in selecting appropriate evaluation metrics depending on the task (e.g., ROC-AUC for classification and MAE for regression). The use of sample weighting in the regression model also illustrated how modifying the training objective can align the model more closely with the economic relevance of the problem.
From a workflow perspective, the project reinforced the importance of clear organization and reproducibility. Structuring the work into separate notebooks (preprocessing, tuning, validation) made experimentation more systematic and easier to track. It also helped maintain a cleaner pipeline, avoid errors, and ensure that results could be replicated and improved iteratively.
Finally, the project emphasized the importance of proper validation and baseline comparison, as well as interpreting model outputs through feature importance analysis to better understand the drivers of predictions.
## 📖 Data
The dataset used in this project is not publicly available, as requested by the course instructor. Therefore, it cannot be shared in this repository.
Part of the methodology implemented in this repository follows the identification strategy proposed in this paper.
The data consists of transactional records from an online European fashion retailer specializing in footwear. It includes customer purchases during a feature construction period (2016–2017), which are used to engineer features, and customer-level revenue in a future period (2018–2019) as the prediction target.
The dataset contains detailed information on transactions, customers, and products (e.g., purchase behavior, discounts, returns, and product characteristics such as brand, type, and material). A key challenge is that a large proportion of customers generate zero revenue in the target period, making the problem inherently sparse and motivating the use of a combined classification–regression approach.
## 🌟 Conclusion
The project shows that combining classification and regression models improves revenue prediction in tabular data. Performance gains were mainly driven by feature engineering, proper validation, and aligning the model with the economic objective.
